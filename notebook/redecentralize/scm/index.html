Git

  • git –distributed-is-the-new-centralized: página oficial del proyecto.

  • tryGit: tutorial interactivo de 15 minutos para conocer los comandos básicos de git desde terminal.

  • Tanto en GNU/Linux como en Windows, se proveen las herramientas git-gui (Git GUI) y gitk (Repository -> Visualize * History).

  • Existen GUIs alternativas, gratuitas y de pago. También soluciones integradas en la shell de Windows, como TortoiseGit. Puede consultarse una lista completa de interfaces y herramientas.

GitHub

  • GitHub Desktop (disponible para Windows y Mac) ofrece una integración completa de credenciales (generación y adición de claves).

NOTA: GitHub Desktop funciona con HTTPS. Hay que añadir un segundo remote para utilizar SSH, ya que si renombramos origin, la aplicación no reconoce el repositorio.

  • If you need to compare it against a old fingerprint you also need to specify to use the md5 fingerprint hashing function.

    ssh-keygen -E md5 -lf .ssh/id_dsa.pub

    2048 MD5:4d:5b:97:19:8c:fe:06:f0:29:e7:f5:96:77:cb:3c:71 (DSA)

  • Además de la gestión de código fuente mediante Git, GitHub ofrece recursos para facilitar la gestión y documentación de los proyectos:

    • Para la documentación se utiliza una sintaxis basada en Markdown, pero con ciertas particularidades. En Markdown Cheatsheet hay una referencia de la sintaxis utilizada en Github.

      • El contenido del fichero REAMDE.md en la raíz del repositorio se muestra en la página principal del proyecto.
      • Se facilita una wiki, accesible a través de github.com/<user|organization>/<repo>/wiki compuesta por múltiples ficheros .md. Además de la posibilidad de editar el contenido a través de la web, en “Adding and editing wiki pages locally” hay instrucciones sobre cómo hacer un git clone de la wiki, y sobre el formato a utilizar a la hora de editar o crear ficheros.

  • Se dispone de un gestor de tareas/bugs, accesible a través de github.com/<user|organization>/<repo>/issues. En Mastering Issues hay una lectura de 10 minutos sobre su uso.

  • Se provee espacio para almacenar una web estática en HTML + CSS, accesible a través de <user|organization>.github.io. Al igual que en el caso de la wiki, el contenido de ésta se gestiona como un repositorio más, tal como se explica en Getting Started with GitHub pages. Adicionalmente, ya que la página GitHub lo utiliza, se soporta el uso de Jekyll, un generador de contenido estático con estructura tipo blog, escrito en Ruby. El artículo Using Jekyll with Pages ofrece más información al respecto.

  • En GitHub Guides hay más guías para comprender el flujo de colaboración promovido en GitHub (basado en ‘forks’ y peticiones de ‘pull’).

  • Bitbucket ofrece servicios similares, y en la versión gratuita permite disponer de repositorios privados, limitando el número de colaboradores a cinco por repositorio. GitHub no establece límite en el número de colaboradores, pero no ofrece la posibilidad de disponer de repositorios privados en el plan gratuito (aunque hay ofertas para estudiantes).

GitLab

GitLab es una alternativa a GitHub, que ofrece servicios similares y además pone el código fuente a disposición de la comunidad para que se puedan realizar instalaciones propias.
